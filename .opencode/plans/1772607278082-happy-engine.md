## Cross-Cutting Concerns

These are domains where multiple features overlap. They inform keybind patterns, grouping, and how features relate тАФ not implementation order.

**Git** тАФ Many features live in the git domain: signs, blame, hunks, conflicts, diff, navigation. They should feel like one cohesive system. Keybinds, commands, and toggles should be consistent across all git features.

**Toggleable UI** тАФ Several features are "turn this on/off": treesitter context, inline blame, zen mode, inlay hints, diagnostics virtual text, etc. Needs a consistent keybind pattern so you don't have to remember per-feature toggles.

**Custom modes** тАФ Some features want a submode: press a key to enter, stay in that mode, press keys without a prefix, exit cleanly. Should feel first-party: show mode in statusline, clean enter/exit. Treewalker is the clear candidate. There might be others тАФ like window resize, diff review тАФ worth exploring/validating later as features are built.

**Text operations** тАФ Surround, split/join, coercion, swap/reorder are all structured text manipulation. They might share a keybind pattern or at least live nearby. `q` textobject should work in surround too (`csq"` = change surrounding any-quotes to double-quotes) for least surprise.

## Shared UI Spaces

Certain screen areas are written to by multiple features. They're not cross-cutting concerns but shared resources тАФ features that touch them should be aware of each other.

**Sign column** тАФ Git signs, diagnostic signs, marks, maybe todo highlights. Must not overlap or get too busy. Everything should fit cleanly.
- Git signs: thin bars, staged vs unstaged visually distinct
- Diagnostics: icons/signs (review what's there, clean up)
- Marks: subtle, nice to have (see #39)
- Todo/fixme: probably highlight in-line text, not signcol тАФ open question

**Statusline** тАФ Multiple features contribute info. Base setup is feature #22. Other features may add components (custom modes, AI, cmdheight 0 extras). Per-filetype overrides needed (Oil, quickfix, undotree, help). Features that touch it should note "Touches: **statusline**."

**Inlay hints / virtual text** тАФ Diagnostics, inline blame, maybe LSP type hints. Can compete for visual space on the same line. Priority/toggle matters.

## Navigation Conventions

Bracket motions follow a consistent pattern:
- `]x` тАФ next
- `[x` тАФ previous
- `]X` тАФ last
- `[X` тАФ first

Applies to: hunks (`h`/`H`), diagnostics (`d`/`D`), quickfix (`q`/`Q`), and any future bracket motions.

All bracket motions should be repeatable with `;`/`,` (see feature #20).

- Just files & grep for now. Other pickers (help, symbols, buffers) can come later or from another tool.
- Nix: fff.nvim has a rust binary. Must work in nix-wrapped nvim (prebuilt download or cargo build).

**Possible plugins:** `fff.nvim` (decided)

### 2тАУ6. Git signs, blame, hunks
Part of: **git**, **toggleable UI** (blame). Touches: **sign column**, **navigation conventions**, **inlay hints/virtual text** (blame).

**Requirements:**
- Thin bar signs in sign column for added/changed/deleted (see: sign column)

### 7. Textobjects
Relates to: **text operations** (`q` ties into surround).

**Requirements:**
- `aF`/`iF` тАФ function definition (`aM`/`iM` alias for method) (treesitter)
- `af`/`if` тАФ function call (`am`/`im` alias) (treesitter)
- `ac`/`ic` тАФ class/module (treesitter)
- `ao`/`io` тАФ block: ruby do..end/`{}`, markdown fenced code block (treesitter)
- `` a` ``/`` i` `` тАФ backtick-delimited: matches nearest code_span OR fenced_code_block (treesitter). Use `o` when you specifically want fenced block.
- `ia`/`aa` тАФ single argument (treesitter)
- `iA`/`aA` тАФ all arguments including parens for `aA` (treesitter)
- `B` тАФ whole buffer (select everything). Note: `gqB` already exists as a dedicated LSP format-whole-file mapping тАФ that's separate, some formatters need the entire file not line ranges. The `B` textobject is for general use with any operator (`dB`, `yB`, etc.).
- `q` тАФ any quotes (nearest surrounding quote pair). Must also work in surround (`csq"` = change surrounding any-quotes to double-quotes).
- `ii`/`ai` тАФ indentation level. `ii` = all contiguous lines at same indent (including blank lines between them). `ai` = same + container lines above/below.
- `ih` тАФ git hunk (comes from git signs, not this feature)
- Degrade gracefully: if treesitter query isn't available for a language, don't error. Fall back or silently do nothing.
- Best-effort per language for function def vs call distinction.
- Maybe: comment block textobject (contiguous comment lines). Key TBD. `@comment.outer` doesn't work well for multi-line `#` comments in Ruby тАФ would need pattern-based approach.
- Maybe: `it`/`at` тАФ HTML/XML tag (built-in vim already has this, but could be enhanced)
- Maybe: bracket motions for textobject types тАФ `]f`/`[f` next/prev function call, `]F`/`[F` first/last function def, `]m`/`[m` method, etc. Follows navigation conventions (`]X`/`[X` for first/last). Repeatable with `;`/`,`. If the textobjects exist, matching motions are least surprise. Add if easy. Treewalker mode may make these redundant.

**Possible plugins:** `nvim-treesitter-textobjects`, `mini.ai`, `nvim-various-textobjs` (can mix)

### 8. Swap / reorder
Relates to: **text operations**.

**Requirements:**
- Swap function/method definitions with sibling above/below тАФ primary use case
- Swap arguments/parameters next/prev тАФ nice to have. Best-effort; Ruby's mix of positional and keyword args may cause edge cases. Good enough is fine.
- Could also apply to array elements, hash entries, etc. тАФ same "swap sibling" concept but no dedicated treesitter queries for those. If it works, great.
- Keybinds TBD. Text operations keybind scheme is unsettled:
  - `gs` is currently surround, can't reuse
  - `gm` is taken (stock: cursor to horizontal middle)
  - Open question: find a home that doesn't clash
  - Forward/backward swap should feel directional (like n/N or ]/[)

**Possible plugins:** `nvim-treesitter-textobjects` (swap), `iswap.nvim`, `mini.move`

### 9. Surround
Relates to: **text operations**. Shares `q` (any quotes) with textobjects.

**Requirements:**
- Add surround: `gsa{motion}{char}` (e.g. `gsaiw'` = surround inner word with single quotes)
- Delete surround: `gsd{char}`
- Replace surround: `gsr{old}{new}`
- `q` works as a target тАФ "any/nearest quotes" (matches textobject `q`)
- Quotes, braces, parens, brackets are the core use case
- Nice to have: tags (`<div>`), function calls. Not used often but nice if the plugin supports it.
- Keybind prefix `gs` тАФ currently in use from existing setup. The whole text operations keybind scheme may change. `g..` is a weird namespace тАФ might move everything under `<leader>` or find a better pattern. To be refined when text operations (surround, swap, split/join, coercion) are considered together.
