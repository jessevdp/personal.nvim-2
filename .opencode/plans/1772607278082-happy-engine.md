## Cross-Cutting Concerns

**Git** — signs, blame, hunks, conflicts, diff, navigation. One domain, many features.

**Toggleable UI** — things you turn on/off. Context, blame, zen, inlay hints, diagnostics, etc. Needs consistent keybind pattern.

**Custom modes** — submodes with first-party feel. Show in statusline, clean enter/exit. Treewalker, window resize, hunk review(?).

**Text operations** — surround, split/join, coercion, swap/reorder. Structured manipulation. Might share keybind pattern. `q` textobject should work in surround too (`csq"` = change surrounding any-quotes to double-quotes).

**Constraint: Multiplexer** — pane navigation and test running depend on terminal multiplexer. Currently zellij, planning to return to tmux. Solution must not be tied to one.

## Implementation Approach

For each feature:
1. Create/edit the config file
2. `vim.pack.add()` if needed
3. Keybindings (stock-adjacent). Not everything is a keybind — commands are fine.
4. Test in real editing
5. `nix flake check`

One feature at a time. Live with it. Adjust. Next.
