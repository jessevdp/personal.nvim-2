# Neovim Config: Road to Daily Driver

## Structure Decision

**Keep current structure.** `lua/config/` for feature configuration (plugin setups, options, keybinds). `plugin/` for standalone behavior autocmds. The `require()` chain in `lua/config/init.lua` is a useful table of contents. No restructuring needed.

New features follow the existing pattern: one file per feature in `lua/config/`, added to the require chain in `lua/config/init.lua`.

---

## Phase 1: Daily-Use Essentials

These are the things you'll reach for every day. Do these first.

### 1. Git signs / hunks / blame
**Plugin:** `lewis6991/gitsigns.nvim` (v2.0 targets nvim 0.11+, nothing built-in replaces it)
**File:** `lua/config/gitsigns.lua`
**Config:**
- Sign column indicators for add/change/delete
- `]h` / `[h` to navigate hunks (stock-adjacent — matches `]q`/`[q` pattern)
- `<leader>hs` stage hunk, `<leader>hr` reset hunk, `<leader>hp` preview hunk
- `<leader>tb` toggle inline blame
- `ih` textobject for selecting hunks (visual + operator-pending)

### 2. fff.nvim (file picker + grep)
**Plugin:** `dmtrKovalenko/fff.nvim`
**File:** `lua/config/picker.lua` (replace current fzf-lua config)
**Config:**
- `<leader>sf` find files, `<leader>sg` live grep
- Remove fzf-lua dependency from lockfile
- `vim.ui.select`: use builtin for now (no extra plugin)
- **Nix note:** fff.nvim downloads a prebuilt binary or builds via rustup. For nix-wrapped nvim, may need to ensure the binary is available or add rustup/cargo to the wrapped PATH. Test both `nix run` and local clone workflows.
- Skip buffer picker for now — see if `<leader>sf` with frecency ranking covers the use case

### 3. Treesitter textobjects
**Plugin:** `nvim-treesitter/nvim-treesitter-textobjects`
**File:** `lua/config/textobjects.lua`
**Textobjects to configure:**
- `af` / `if` — function (outer/inner)
- `ac` / `ic` — class/module
- Custom (non-treesitter):
  - `B` — whole buffer (`ggVG` equivalent)
  - `q` — any quotes (nearest quote pair)
  - Ruby: `o` — ruby block (do..end / { })
- **Note on `gc`:** Built-in comment operator in nvim 0.10+. `gcc` comments a line, `gc{motion}` comments a range. Don't override it. If you want a "comment block" textobject, `gcip` (comment inner paragraph) already works. Skip the custom `C`/`gc` textobject.

### 4. leap.nvim (2-char motions)
**Plugin:** `ggandor/leap.nvim`
**File:** `lua/config/leap.lua`
**Config:**
- `s` in normal/visual/operator-pending → `<Plug>(leap)` (bidirectional)
- Minimal config, try stock behavior first
- Consider `repeatable-move.nvim` later to make `;`/`,` repeat leap + hunk nav + any bracket motion

### 5. Treesitter context
**Plugin:** `nvim-treesitter/nvim-treesitter-context`
**File:** `lua/config/treesitter.lua` (add to existing)
**Config:** Defaults are fine. Shows function/class context at top of window.

## Phase 2: Quality of Life

Do these once Phase 1 is solid.

### 6. Git merge conflicts
- Conflict marker highlighting (can be done with simple `matchadd` or a plugin like `akinsho/git-conflict.nvim`)
- `git jump merge` → populate quickfix via `cgetexpr` (small utility function)
- `git jump diff` bindings

### 7. Treewalker mode
**Plugin:** `aaronik/treewalker.nvim`
- Structured movement through AST nodes
- Possibly wrap in a hydra-like "mode" for rapid traversal
- Keybinds TBD

### 8. Treesitter split/join
**Plugin:** `Wansmer/treesj` or `echasnovski/mini.splitjoin`
- Toggle between single-line and multi-line forms

### 9. Key hints
**Plugin:** `folke/which-key.nvim`
- Shows available keybindings after pressing leader or any prefix
- Helps learn the config

### 10. Quick wins
- **Jbuilder filetype:** `vim.filetype.add({ extension = { jbuilder = "ruby" } })` in `ftplugin/` or `lua/config/`
- **Zsh edit-command-line:** Set filetype to zsh for the temp file (check `$EDITOR` buffer name pattern)
- **Whitespace fixing:** `vim.api.nvim_create_autocmd("BufWritePre", { callback = strip_trailing_whitespace })`

### 11. Statusline
- Roll a simple one with `vim.opt.statusline` (mode, branch via gitsigns, filename, diagnostics, position)
- Or use `echasnovski/mini.statusline` for minimal setup
- Low effort, high visual payoff

### 12. Folds
- Already using treesitter foldexpr. May need LSP fallback.
- Consider `kevinhwang91/nvim-ufo` if the built-in fold UX isn't enough

## Implementation Order

For each item in Phase 1:
1. Add plugin to `vim.pack` in the config file
2. Run `vim.pack.update()` to lock the revision
3. Configure with keybindings
4. Test
5. Run `nix flake check`
