## Phase 1: Daily-Use Essentials

Build one at a time. Evaluate. Tweak. Next.

### 1. File Picker & Grep
Replace fzf-lua. Fast file finding with frecency. Live grep.
- `<leader>sf` find files, `<leader>sg` live grep
- `vim.ui.select`: use builtin for now
- **Nix concern:** if the tool has a native binary, ensure it works in nix-wrapped nvim

**Options:** `fff.nvim` (opinionated, rust backend, frecency)

**File:** `lua/config/picker.lua` (replace current)

### 2. Git Signs / Hunks / Blame
Sign column indicators. Navigate hunks `]h`/`[h`. Stage/reset hunks. Toggle inline blame. Hunk textobject (`ih`).

**Options:** `gitsigns.nvim` (v2.0, nvim 0.11+)

**File:** `lua/config/git.lua`

### 3. Textobjects
Mix of treesitter-based and custom.

**Desired textobjects:**
- `aF`/`iF` — function definition (`M` alias for method) (treesitter)
- `af`/`if` — function call (`m` alias for method) (treesitter)
- `ac`/`ic` — class/module (treesitter)
- `o` — ruby block, do..end / `{ }` (treesitter)
- Markdown: code fences (treesitter)
- `B` — whole buffer
- `q` — any quotes (nearest quote pair)
- `ih` — git hunk (from git signs plugin)

**Note on `gc`:** Built-in comment operator since nvim 0.10. `gcc` comments a line, `gc{motion}` comments a range. Don't override. `gcip` already comments a paragraph.

**Options:**
- `nvim-treesitter-textobjects` — treesitter queries for function, class, etc.
- `mini.ai` — framework for any `a`/`i` textobject. Treesitter integration via `gen_spec.treesitter()`. Good home for custom ones (`B`, `q`).
- `nvim-various-textobjs` — 30+ pattern-based textobjects. Pick what you need.
- Can mix these.

**File:** `lua/config/textobjects.lua`

### 4. Fast Motions (2-char jump)
Quick cursor movement to any visible location. Try 2-char motions first. May miss flash's remote actions — start simple.

**Options:**
- `leap.nvim` — 2-char, minimal
- `flash.nvim` — labels on search, treesitter selection, remote. More complex.

**File:** `lua/config/motions.lua`

### 5. Motion Repeat
Make `;`/`,` repeat any `]`/`[` motion (hunks, diagnostics, quickfix, treesitter moves). Treesitter-textobjects dropped this feature.

**Options:**
- `demicolon.nvim` — auto-repeats all `]`/`[` motions with `;`/`,`
- `vim-remotions` — more general
- Manual wrapper

**File:** `lua/config/motions.lua` or `plugin/repeat_motions.lua`

### 6. Treesitter Context
Show current function/class at top of buffer while scrolling. **Toggleable.**

**Options:** `nvim-treesitter-context`

**File:** add to `lua/config/treesitter.lua`

### 7. Markdown
**a. Wrapping** — Auto-wrap in markdown (except code blocks). Remap `j`/`k` to `gj`/`gk`.
- Mostly `vim.opt_local` in `ftplugin/markdown.lua`: `wrap`, `linebreak`, `breakindent`.

**b. In-buffer rendering** — Headings, links, code fences, checkboxes rendered visually.
- **Options:** `markview.nvim`, `render-markdown.nvim`

**File:** `ftplugin/markdown.lua` + `lua/config/markdown.lua`

### 8. Buffer Management
Close buffers without destroying window layout. `<leader>x` to quickly close current buffer.

**Options:** `bufdel.nvim`, `mini.bufremove`, or hand-rolled (`:bp | bd #`)

**File:** `lua/config/buffers.lua` or `plugin/bufdelete.lua`

## Cross-Cutting: Key Hints + Custom Modes

1. **Key hints** — show available keys after pressing a prefix. Toggleable.
2. **Custom modes** — "treewalker mode," window resize mode, etc. Stay in a submode pressing keys without re-pressing the prefix.

**Options:**
- `which-key.nvim` — key hints + basic hydra via `show({ loop = true })`
- `hydra.nvim` — dedicated submodes. More powerful (colored heads, hints, hooks). No general key hints.
- Both together: which-key for discoverability, hydra for submodes.

**File:** `lua/config/keyhints.lua` and/or `lua/config/modes.lua`

## Implementation Approach

For each feature:
1. Create/edit the config file
2. `vim.pack.add()` if needed
3. Keybindings (stock-adjacent)
4. Test in real editing
5. `nix flake check`

One feature at a time. Live with it. Adjust. Next.
